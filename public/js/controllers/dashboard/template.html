Track Lab

What is Track Lab?

Track Lab is an opensource self-hosted data aggregation API for developers.

It's a free and lite alternative to services like MixPanel, Customer.io, etc.

Authority

Made by Iliya Garakh during the Koding Hackathon 2016.

How it works

There are two general entities: people and events.

People is data about users, it's like user table but stored in TrackLab.

Event is really free object, you can store payments, hits, authorization etc info. Every event is connected with user issued this event.

Track Lab gathers the information and runs Triggers and Handlers. Trigger is a PHP class stored in /app/trigger/ with check method (all connected data are passed via params). This way you can implement any conditions, for instance AgeTrigger checks age field from incoming people info. If trigger's check method returns true then the system fires associated handler. Handler is a PHP class with method run stored in /app/handler/. E.g. EmailHandler sends emails to users.

People triggers work the same way but only once for associated user.

Feel free to create your own triggers. Store them in /app/trigger/event/project-name/ and /app/trigger/people/project-name/. Check out the source code to figure out how it works in details.

You can automatize sending hello e-mail to new users, gathering any statistics, sending notificaitions to Slack, and any other scenario which you can dream.

How to install

You need to have installed:

  • PHP with Phalcon PHP Framework 2
  • MongoDB

Installation:

git clone https://github.com/garakh/tracklab

cd tracklab/public

bower install

API

Send POST request to /gate

People

Fields:

  • type = people
  • email — (string) user's email
  • project — (string) project code (e.g. demo). Needs to namespace your data
  • data — (json) any object to save

Event

Fields:

  • type = event
  • name — (string) event's name(e.g. payment, signin)
  • project — (string) project code (e.g. demo). Needs to namespace your data
  • data — (json) any object to save

Don't care about preparations, just send data and everything will be created automatically (collections, projects, etc)

Example

                    type = event
                    name = signin
                    project = mysite
                    data = {"from" : "landing"}
